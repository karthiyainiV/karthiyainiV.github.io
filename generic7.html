NYC Apartment Data Analysis Using SQL

The "NYC Apartment Data set" project utilizes Power BI to analyze and visualize data related to apartments in New York City. The project involves various SQL queries executed on the dataset to derive meaningful insights for visualization purposes. The following queries are used in the project: Selecting all data from the "Nycapartment$" table: This query retrieves all the data from the "Nycapartment$" table, which serves as the main data source for the project. Filtering rows where the price is null: This query filters the dataset to include only rows where the price is null, allowing for a separate analysis of missing data. Adding a "Year_wise_review" column: This query adds a new column named "Year_wise_review" to the "Nycapartment$" table, which contains the year extracted from the "last_review" column. Dropping the "review_as_per_year" column: This query drops the "review_as_per_year" column from the "Nycapartment$" table. Grouping by "neighbourhood_group" and counting occurrences: This query groups the data by the "neighbourhood_group" column and counts the occurrences of each group, providing insights into the distribution of apartments across different neighborhood groups. Joining and aggregating data for room types: This query joins and aggregates data for room types, calculating the average price, maximum reviews per month, and maximum reviews for each room type. The results are sorted by average price, maximum reviews, and maximum reviews per month. Joining and aggregating data for room types (averages): This query joins and aggregates data for room types, calculating the average price, average reviews per month, and average reviews for each room type. The results are sorted by average price, average reviews, and average reviews per month. Calculating average price per night for each room type: This query calculates the average price per night for each room type using a subquery, grouping the data by room type. Counting total occurrences with availability of 365 days: This query counts the total occurrences of each neighborhood group where the availability of apartments is 365 days. Calculating average review percentage per year: This query calculates the average review percentage per year by grouping the data by the "Year_wise_review" column and calculating the average of "reviews_per_month". Analyzing apartment details for 2019: This query creates a common table expression (CTE) named "Apartment_details_2019" that selects specific details from the dataset for the year 2019 and apartments with availability of 365 days. It calculates the average price per night for each neighborhood group. Comparing average room prices on multiple nights with single-day prices: This query compares the average room prices on multiple nights with the price of a single day, grouping the data by "neighbourhood_group". It calculates the average price per night and the average price for a single day. These queries provide insights into various aspects of the NYC apartment dataset, including neighborhood group distribution, average prices, availability, and review trends. The results of these queries can be visualized using Power BI to create interactive and visually appealing dashboards, reports, and charts that enable users to explore the data and gain valuable insights into the NYC apartment market.
